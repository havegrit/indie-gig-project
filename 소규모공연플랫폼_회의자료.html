1핵심 가치

권장 한 줄 "흩어진 인디 공연을 한 곳에서 발견하고 놓치지 않게 한다."

가치 분해 — 3개 축

  • 발견(Discovery) — 인스타·트위터·커뮤니티에 흩어진 정보를 구조화해 검색·필터링 가능하게.
  • 알림(Notification) — 티켓 오픈·관심 아티스트 공연을 놓치지 않게 푸시.
  • 큐레이션(Curation) — 단순 어그리게이션이 아닌 권위 있는 픽 (RA Picks 모델).

2해결할 수 있는 문제

👥 팬 측 문제

  • 공연 정보가 인스타·트위터·커뮤니티에 흩어져 다채널 모니터링 필요
  • 예매처가 인터파크·멜론·예스24·티켓링크로 분산 → 티켓 오픈 추적 어려움
  • 스크린샷·사진첩으로 우회 관리하는 비효율
  • 좋아하는 아티스트의 신규 공연을 놓침
  • "이번 주말 홍대 공연 뭐 있지?" 자연어 탐색 불가

🎤 공급자 측 문제 (밴드·공연장)

  • 평균 연소득 901만원(국민 평균의 41%) — 도달이 곧 생계
  • 홍대 라이브클럽 70%+ 폐업·휴업 (2021 기준)
  • 임대료 월 500만~2,000만원, 일반음식점 등록으로 지원 사각
  • 신인 등용 채널 부재 — 대학축제·방송은 대형 기획사 독점
  • 인스타·틱톡 알고리즘에 전적으로 의존한 자체 홍보

시장 구조적 문제

  • KOPIS는 정식 등록 공연장 위주 → 일반음식점 등록 라이브클럽 다수 누락
  • 대형 예매처는 인디 카테고리에 무관심
  • K-팝과 인디 양극화 심화

4주요 고객층

논의 전제 "인디 공연 팬"은 너무 넓다. 안에 사실상 세 명의 다른 사람이 있고, 각 페르소나는 다른 제품을 필요로 한다. 세 명을 동시에 만족시키는 단일 UX는 거의 불가능하다. 회의에서 1차 페르소나를 한 명 고른다.

4-1. 페르소나 3유형 비교

구분A. 헤비 팬B. 라이트 팬C. 입문자
관람 빈도 월 3회+ 분기 1~2회 연 1~2회 또는 0회
행동 패턴 좋아하는 밴드 10팀+ 보유. 인스타·트위터·디시 매일 모니터링. 스크린샷·캘린더 관리 직접 친구 따라가거나 "뭐 재밌는 거 없나" 검색. 특정 밴드 충성도 낮음 인디 씬에 호기심. 어디서 시작할지 모름. "홍대 가면 뭐 봐?"
진짜 페인포인트 티켓 오픈 놓침. 좋아할만한 신인 밴드 발견 어려움 "오늘/주말 뭐 볼만한 거 있지?"가 안 풀림 신뢰할 수 있는 가이드 부재
필요한 제품 알림 + 유사 아티스트 추천 캘린더 + 큐레이션 + 친구 시그널 에디토리얼 + 입문 가이드 + 추천
리텐션 강도 ★★★ 매주 앱 열 이유 있음 ★ 분기 1회는 앱 설치 안 함 ★ 일회성 이탈 위험
마케팅 비용 낮음 (커뮤니티 입소문) 중 (검색 광고·SEO) 높음 (브랜드 인지 필요)

4-2. 권장 1차 페르소나 — A. 헤비 팬

한 줄 "월 3회 이상 인디 공연을 직접 챙겨 보는 20~30대, 좋아하는 밴드 10팀+ 보유, 인스타 스토리·DM·디시·트위터를 매일 정보 채널로 쓰는 사람."

선정 근거

  1. 시장 진입 비용이 가장 낮다. 이미 자기 동기로 인디 공연을 찾고 있어 가치 제안이 짧은 한 문장으로 끝난다 ("티켓 놓침 / 스크린샷 헬"). 마케팅 비용 거의 0.
  2. 데이터 부트스트랩이 가능하다. 헤비 팬이 "내가 좋아하는 밴드" 50개를 등록해주면, 그것 자체가 인기 아티스트 시그널 · 신인 추천 데이터 · 공연 우선순위가 된다. 라이트 팬·입문자는 이 데이터 위에서 후행으로 만족시킬 수 있다.
  3. 리텐션이 가장 강하다. 헤비 팬은 매주 앱을 열 이유가 있다. 라이트 팬은 분기 1회 열려고 앱을 설치하지 않는다.
  4. NPS 전파력. 인디 씬 헤비 팬은 좁고 끈끈한 커뮤니티. 트위터·디시·커뮤니티에서 입소문 폭발력이 크다.
  5. 확장 방향이 자연스럽다. 헤비 팬용 큐레이션·데이터가 쌓이면 "Top 10 이번 주 공연" 같은 라이트 팬용 진입점을 만들기 쉽다. 반대 방향(라이트→헤비)은 불가능.

위험 / 반론

  • TAM(잠재 시장)이 작다. 월 3회+ 인디 헤비 팬은 추정 수만 명 단위. 단독 사업으로 한계 — 확장 시나리오 필수.
  • 헤비 팬은 이미 자기 방법이 있다. 인스타·메모·캘린더로 직접 관리. 대체할 만큼의 가치를 줘야 함 (≒ 통합 알림 + 추천이 핵심).
  • "이 앱 없어도 공연은 본다" — 가치는 편의성 + 발견 보너스. 절대 의존은 아님.

4-3. 2차 타겟 (6~12개월 후 확장)

  • B. 라이트 팬 — 헤비 팬이 만든 데이터 기반 "이번 주 Top 10" UX로 유입
  • C. 인디 입문자 — Spotify·멜론 청취 이력 + 에디토리얼 큐레이션으로 유입
  • 지방 거주자 — 수도권 방문 시 묶어보기, 지역 페스티벌 연계

4-4. 공급자 측 (별도 트랙)

  • 라이브클럽 / 공연장 운영자 (홍대·망원·을지로 등)
  • 인디 밴드·소속사·기획자 (음레협 산하 3,168팀)
  • 페스티벌·라이브클럽데이 같은 기획 IP

4-5. MZ 행동 트렌드 (헤비 팬과의 정합성)

  • "디토 소비" — 신뢰하는 큐레이터 따라 구매 (헤비 팬은 자신이 곧 큐레이터, 추종자를 만들고 싶어함)
  • "가심비" / "로코노미" — 동네·취향 중심 소비 (홍대·망원·을지로 권역과 정합)
  • N차 관람 — 인디 팬덤은 충성도 매우 높음 (헤비 팬의 정의 자체)

4-6. 회의에서 결정할 것

  1. 1차 페르소나 = A. 헤비 팬으로 확정 / 아니면 B 또는 C 중 어떤 근거로?
  2. "헤비 팬" 정의를 더 좁혀야 하나? (예: "월 3회+ AND 특정 장르(인디 락/일렉트로닉/재즈) 중심" 등)
  3. 헤비 팬을 어떻게 첫 100명 확보할 것인가? (커뮤니티 시드 / 라이브클럽 협업 / 인플루언서)
  4. 헤비 팬 단독으로 TAM 한계가 명확함 — 6개월 시점의 확장 시나리오를 지금 합의할지, 검증 후 결정할지?

5관련 사회 문제

5-1. 인디 뮤지션 빈곤

  • 음악 분야 예술인 평균 연소득 901만원 (국민 평균의 41.3%)
  • 76%가 월 100만원 미만, 절반이 부업

5-2. 라이브클럽 폐업 / 젠트리피케이션

  • 홍대 라이브클럽 70%+ 폐업·휴업 (2021)
  • 브이홀·무브홀·살롱 노마드·퀸라이브홀 등
  • 일반음식점 등록 → 정부 지원 배제

5-3. 발견 인프라 붕괴

  • 네이버 OnStage 종료 (2023)
  • EBS 헬로루키 중단 (2023)
  • 대중과의 접점 절대 부족

5-4. K-팝과 인디의 디커플링

  • K-팝 해외 매출 추정 1조 2,377억원
  • 같은 시기 인디 라이브 관람률 27.5% → 15.4%
  • 시장 회복은 대형 K-팝 콘서트만 흡수
"인디 뮤지션의 수는 계속 증가하나 공연장 감소, 인건비 상승으로 제작 활동이 어려워지고 있다. 생활비에 쫓겨 아르바이트하며 여가 시간에 음악을 생산하다 보니 음악 퀄리티가 낮아질 수밖에 없다." — 윤동환 음레협 회장

5-5. 청년·로컬 문화 활성화 트렌드

  • 서대문구 "청년음악도시 신촌" 사업 — 정책 트렌드 시그널
  • "로컬 컬처 활성화" 정부 키워드 부상

한계 = 후속 서비스의 차별화 기회

  1. 데이터 수집 자동화 부재 — Chrome 확장(사용자 9명)이 유일한 보조 도구, 수기 입력 의존. → LLM 기반 포스터 OCR + 인스타 자동 수집 파이프라인으로 차별화 가능
  2. 앱이 얇음 — Expo로 빠르게 만든 정보 조회 클라이언트, 푸시·캘린더·위시리스트·팔로우 미구현. → 앱 네이티브 경험(푸시·위치 기반·캘린더 연동)이 큰 차별화 축
  3. 스토어 검색 노출 약함 (ASO 부재) — 사용자가 DM으로 링크 요청. → 마케팅·온보딩·SEO/ASO 투자가 즉시 효과
  4. 1인 운영 리스크 — 1세대도 운영자 사정으로 종료된 전례. 팀 운영·자금 조달 시 안정성에서 우위
  5. 수익 모델 미정 — 후원·자체 티켓팅 초기 단계. 광고/Pro 구독/B2B SaaS 등 본격 BM 설계 시 추월 여지
  6. 커뮤니티 기능 부재 — 공식 디스코드/카톡방 없음, SNS는 일방향 공지 중심. → 팬 커뮤니티·세트리스트·후기가 차별화 포인트

8종합 권고 — 회의에서 결정할 것

  1. 핵심 가치 한 줄 합의 — 발견 / 알림 / 큐레이션 중 어디에 무게를 둘 것인가
  2. MVP 범위 합의 — 지역(홍대 한정?) × 장르(인디 락? 일렉트로닉 포함?) × 시간 범위
  3. 데이터 소스 우선순위 — 공연장 파트너십 / UGC / 인스타 API / 스크래핑(법적 리스크) 중 선택
  4. BM 우선순위 — 예매 어필리에이트 / 자체 티켓팅(Dice 모델) / 광고 / B2B 공급자 SaaS
  5. 차별화 한 가지만 — RA식 에디토리얼 / Dice식 정찰제 티켓팅 / LiveFans식 세트리스트 DB / 한국형 알림 통합
